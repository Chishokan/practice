# ウィンターカップへの道

高校バスケ部のチーム育成シミュレーション。
ブラウザだけで動きます。ビルド不要、サーバー不要。

`index.html` を開けば遊べます。

---

## 遊び方

4月からの36週間、8人の部員を育てて、12月のウィンターカップ優勝を目指します。

- **練習** … 部員ごとに毎週メニューを指示する。伸びと疲労は必ず釣り合っている
- **試合** … 5人を選び、戦術を選び、1Qずつ戦う。交代と切り札の使い所が勝負
- **大会** … 6月の県大会、8月の全国（最難関）、12月のウィンターカップ
- **ストーリー** … 毎月1話。育て方と大会の結果で分岐する

セーブはブラウザに自動保存されます（週が変わるたび）。

## ファイルの構成

| ファイル | 役割 |
|---|---|
| `index.html` | 画面の骨組み。中身は全部JSが埋める |
| `game.js` | 選手データ、週の進行、練習、セーブ |
| `events.js` | 学校行事、メインストーリー、選択式イベント、特殊技能 |
| `match.js` | 試合のシミュレーション、戦術、大会 |
| `history.js` | 戦歴 |
| `profile.js` | 三宅の手帳（選手詳細）、今週のひとこと |
| `ending.js` | エンディング5分岐 |
| `style.css` | 見た目。末尾にスマホ用 |
| `images/` | キャラクター10人、エンディング5枚、エンブレム4校 |

読み込む順番に意味があります（`game.js` が土台、他はその上で動く）。

## 設計の骨格

**パラメータは2層構造**

練習で伸ばせるのは6つの基礎パラメータ（パワー・スピード・スタミナ・ジャンプ・テクニック・バスケIQ）だけ。
試合で使う5つの能力（シュート力・ドリブル力・パス力・ディフェンス力・リバウンド力）は、
そこから重み付き平均で計算されます。

```js
{ key: "defense", weights: { speed: 0.30, stamina: 0.25, power: 0.25, iq: 0.20 } }
```

「シュート力を上げたい」が「テクニックとIQを鍛える」に翻訳されるのが、このゲームの中心です。
重みの合計を必ず 1.0 にしてあるので、結果は自動的に 0〜100 に収まります。

**相手校も選手と同じ形**

対戦校も6つの基礎パラメータだけを持ちます。だから味方と相手で同じ計算関数がそのまま通ります。

**状態は2つに集約**

`players` と `state` の2つだけ。だからセーブは丸ごとJSON化するだけで済みます。

## バランス調整をするなら

数字は全部1箇所に集めてあります。

| 変えたいこと | 場所 |
|---|---|
| 疲労・成長効率・ケガの確率 | `game.js` の「疲労のルール」4関数 |
| 練習メニューの伸びと負荷 | `game.js` の `MENUS` |
| 試合能力の計算式 | `match.js` の `DERIVED_STATS` |
| 戦術の相性 | `match.js` の `MATCHUP` |
| 相手校の強さ | `match.js` の `TOURNAMENTS` |

変更したら、必ずシミュレーションで確かめること。
画面を見ただけでは分からない壊れ方をします（実際、決勝が同点で敗戦になるバグを
数百試合回して見つけました）。

macOS なら Node なしで確認できます。

```bash
/System/Library/Frameworks/JavaScriptCore.framework/Versions/A/Helpers/jsc テスト.js
```

## 画像

`images/` に置くだけで反映されます。無くても遊べます（頭文字が出ます）。
生成用のプロンプトは `画像プロンプト.md` に全部あります。

新しく作った画像は、これで取り込めます。

```bash
./画像を取り込む.sh
```

エンディング絵のリサイズと、エンブレム4校の切り出しをやります。
